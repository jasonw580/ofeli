Detailed Description

The class ActiveContour contains the implementation of the Fast-Two-Cycle (FTC) algorithm of Shi and Karl as it describes in their article "A real-time algorithm for the approximation of level-set based curve evolution" published in IEEE Transactions on Image Processing in may 2008.

This class should be never instantiated because the function to calculate the external speed Fd is too general. The 3 classes ofeli::ACwithoutEdges, ofeli::ACwithoutEdgesYUV and ofeli::GeodesicAC, inherit of the class ActiveContour, with for each class it own implementation of this function.

Parameters:
width1Image width, i.e. number of columns. Passed to width.
height1Image height, i.e. number of rows. Passed to height.
hasEllipse1Boolean to choose the shape of the active contour initialization, true for an ellipse or false for a rectangle.
init_width1Width of the shape divided by the image width.
init_height1Height of the shape divided by the image height.
center_x1X-axis position or column index of the center of the shape divided by the image width subtracted by 0.5.
center_y1Y-axis position or row index of the center of the shape divided by the image height subtracted by 0.5.
To have the center of the shape into the image : -0.5 < center_x1 < 0.5 and -0.5 < center_y1 < 0.5.
hasSmoothingCycle1Boolean to have or not the curve smoothing, evolutions in the cycle 2 with an internal speed Fint. Passed to hasSmoothingCycle.
kernel_length1Kernel length of the gaussian filter for the curve smoothing. Passed to kernel_length.
sigma1Standard deviation of the gaussian kernel for the curve smoothing. Passed to sigma.
hasCleaning1Boolean to eliminate or not redundant points in Lin and Lout lists. Passed to hasCleaning.
Na1Number of maximum iterations the active contour evolves in the cycle 1, external or data dependant evolutions with Fd speed. Passed to Na_max.
Ns1Number of maximum iterations the active contour evolves in the cycle 2, curve smoothing or internal evolutions with Fint speed. Passed to Ns_max.
